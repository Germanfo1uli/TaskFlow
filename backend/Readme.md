## Работа с Docker Composer (запуск контейнеров и сборка образов)

#### Для запуска стека контейнеров были написаны скрипты, облегчающие сборку проекта и реализующие быстрый запуск по 1 кнопке (пока только в dev)

#### Для начала скопировать .env файл из .env.example, опционально заполнить своими значениями

#### Файлы с расширением .ps1 запускать только в павершелле винды (иначе капут)

## Основа: Запуск всех контейнеров (контейнеры не будут пересобраны, но если образы не будут найдены, то создадутся новые)
#### .\start.ps1

#### После успешного запуска всех контейнеров можно продолжать работу в Docker Desktop (обьективно удобнее)\

#### Там же можно указывать нужные контейнеры для быстрого запуска/выключения

## Остальные команды:
#### Пересоздание контейнеров (например, в случае изменения исходного кода)
#### .\start.ps1 -Rebuild

#### Полное пересоздание контейнеров (использовать в крайнем случае, уничтожает сохраненные данные в БД)
#### .\start.ps1 -Rebuild -ForceRecreate

#### Просмотр логов (выводит последние логи, можно просматривать в Docker Desktop)
#### docker compose -f docker-compose.dev.yml logs -f user-service

#### Пересборка отдельного образа (не трогает уже существующие)
#### docker compose -f docker-compose.dev.yml up -d --build user-service

## Порты:
#### PgAdmin: http://localhost:5050 (попросит авторизоваться по данным из .env)
#### Swagger UI: http://localhost:8001/swagger-ui/index.html#/
#### API User Service URL: http://localhost:8001/api
